So'rov Oqimi (Request Flow)

Frontend dan Backend gacha so'rov qanday yo'nalish bosib o'tadi

1

👤 Foydalanuvchi login qiladi

Frontend: POST /api/v1/auth/token/ — username va password yuboradi. Yoki POST /api/v1/auth/phone-login/ orqali telefon raqam bilan.

2

⚡ Gateway so'rovni qabul qiladi

FastAPI Gateway :8080 portida so'rovni qabul qilib, HTTPX async client orqali Django Backend ga :8000 proxy qiladi. Host headerini olib tashlaydi.

3

🐍 Django JWT token qaytaradi

CustomTokenObtainPairSerializer ishlatiladi. Token payload ga role va full_name qo'shiladi. Access (60 min) va Refresh (7 kun) tokenlar qaytariladi.

4

🔐 Keyingi so'rovlarda Authorization header

Frontend: Authorization: Bearer <access_token> header bilan so'rov yuboradi. JWT middleware tokenni tekshiradi va foydalanuvchini aniqlaydi.

5

✅ Permission tekshiruvi

ViewSet permission klasslari (IsAdmin, IsTeacher, IsStudent va h.k.) request.user.role ga qarab ruxsatni tekshiradi. Ruxsat bo'lmasa 403 Forbidden.

6

📦 Ma'lumot qaytariladi

DRF Serializer orqali queryset serialize qilinadi, pagination (20 ta/sahifa) qo'llaniladi va JSON formatda response qaytariladi. Celery tasklar asinxron ishlaydi.

Error Handling

Tizimda ishlatilgan barcha xatolik turlari va ularning JSON formati

400
Bad Request — Validatsiya xatosi
Serializer.is_valid(raise_exception=True) avtomatik chaqiriladi. Maydon xatolari field-level ko'rsatiladi.
{
  "username": [
    "Bu maydon talab qilinadi"
  ],
  "password": [
    "Kamida 8 belgi"
  ]
}
401
Unauthorized — Autentifikatsiya yo'q
JWT token yo'q yoki muddati o'tgan. DRF JWTAuthentication avtomatik qaytaradi.
{
  "detail": "Authentication credentials were not provided.",
  "code": "not_authenticated"
}

// Token muddati o'tgan:
{
  "detail": "Token is invalid or expired",
  "code": "token_not_valid"
}
403
Forbidden — Ruxsat yo'q
Foydalanuvchi autentifikatsiyadan o'tgan, lekin kerakli rol/huquq yo'q. Permission class qaytaradi.
{
  "detail": "You do not have permission to perform this action."
}

// Recovery custom xato:
{
  "detail": "Bu vazifa sizga tegishli emas"
}
{
  "detail": "Ruxsat yo'q"
}
404
Not Found — Topilmadi
Ob'ekt topilmadi. DRF get_object() avtomatik qaytaradi. Ba'zi custom viewlarda ham ishlatiladi.
{
  "detail": "Not found."
}

// Custom (grants):
{
  "detail": "Faol semestr topilmadi"
}
503
Service Unavailable — Backend yetishmaydi
Gateway backend ga ulana olmagan. HTTPX RequestError ushlanganda qaytariladi. Faqat Gateway darajasida.
// /health/backend:
{
  "status": "unreachable",
  "detail": "Backend service is not available"
}

// Proxy xato:
{
  "detail": "Backend unavailable: ..."
}
200
Muvaffaqiyatli Javoblar
Standart DRF pagination formati. Barcha list endpointlarda (20 ta/sahifa).
{
  "count": 42,
  "next": "http://.../api/v1/...?page=2",
  "previous": null,
  "results": [
    // ob'ektlar ro'yxati
  ]
}

Grant Scoring Tizimi

Talabalar grant ballarini hisoblash formulasi (jami 100 ball + bonus)

40
Maksimal: 40 ball
📋 Davomot

Maksimal: 20 ball

🏆 Faoliyat

10
Maksimal: 10 ball
⚠️ Jarima (ayiriladi)
-20
Maksimal: -20 ball
✅ Tiklanish (qo'shiladi)
+10
Maksimal: +10 ball
💼 Ish joyi (bonus)
+10
Internship: 0-5, Part-time: 5-7, Full-time: 7-10
Formula: base_total = min(academic + attendance + assignment + activity + tutor + discipline, 100)
final_score = max(base_total + penalty + recovery + employment, 0)

GPA shart: gpa_percentage ≥ 80% bo'lmasa gpa_eligible = False → grant bloklanishi mumkin.
Grant ajratish: 3 bosqich — Rating saralash → Suhbat → Yakuniy qaror (approved / waitlist / rejected)
